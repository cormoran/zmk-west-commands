## What you get

Add one entry to your west manifest and four commands appear in `west`. Two make
building `zmk-config` firmware effortless; two run your module's tests **without a
single piece of hardware**.

<CardGrid stagger>
  <Card title="west zmk-build" icon="seti:cpp">
    A `west build` wrapper that reads your `build.yaml`, picks targets
    interactively, flashes, and ships handy shortcuts (`--reset`,
    `--debug-jlink`, `--vscode`).
  </Card>
  <Card title="west zmk-test" icon="approve-check">
    Run ZMK's native-posix unit tests with the environment wired up for you —
    point it at a directory and pass `-m .` to include your own module.
  </Card>
  <Card title="west zmk-renode-test" icon="rocket">
    Boot a real firmware ELF in the Renode emulator and run a boot + Studio
    smoke test — BLE pairing or UART, no J-Link, no board.
  </Card>
  <Card title="west zmk-ble-test" icon="random">
    Run split + Studio-over-BLE integration tests under BabbleSim, asserting
    output against checked-in snapshots. Pure software.
  </Card>
</CardGrid>
